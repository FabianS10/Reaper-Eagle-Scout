# Reaper Eagle Scout**Public Web Opportunity Intelligence for startups, founders, and small vendors.**> **Move before the market does.**Reaper Eagle Scout is a full-stack web intelligence platform that helps small teams discover public tenders, RFPs, procurement notices, grants, and market opportunities from the live web.Large companies have analysts, procurement teams, market intelligence departments, and expensive enterprise platforms. Small startups usually have browser tabs, manual searches, and missed deadlines.Reaper Eagle Scout gives small teams their own opportunity reconnaissance engine.It uses **Bright Data**, **Python/FastAPI**, and **React** to discover global public opportunities, enrich messy web data, classify sources, estimate deadline urgency, and rank opportunities by strategic value.---## Table across public web sources such as:* Government procurement portals* Tender databases* RFP listings* Public contract notices* Grant opportunities* International procurement platforms* Market opportunity signals* Search engine result pagesInstead of only returning raw links, Reaper Eagle Scout structures each result into a candidate opportunity and enriches it with:* Source entity* Country or region* Opportunity type* Sector* Deadline urgency* Confidence score* Strategic relevance score* Source evidence* Original URLThe goal is to help founders and small teams answer one practical question:> **What opportunity should I act on before it disappears?**---## ProblemPublic business opportunities are scattered across the web.A startup founder may need to monitor:* Government tender portals* Procurement databases* International development organizations* Public RFP pages* PDFs and notice documents* Competitor and market signals* Regional grant opportunities* Search engine resultsThis creates several problems:1. **Discovery is manual**   Founders lose time searching across many websites.2. **Information is fragmented**   Deadlines, budgets, requirements, and source entities are often buried in inconsistent formats.3. **Small teams lack intelligence infrastructure**   Large enterprises can afford analysts and procurement monitoring tools. Small vendors usually cannot.4. **Opportunities expire quickly**   Missing a deadline means losing a possible contract, grant, or strategic opening.5. **Raw web data is not decision-ready**   A search result does not tell the user whether an opportunity is relevant, urgent, credible, or worth pursuing.---## SolutionReaper Eagle Scout acts as a public web opportunity scout.The system:1. Accepts a user search query.2. Uses Bright Data-powered web discovery to search the public web.3. Retrieves candidate sources.4. Parses and enriches the discovered data.5. Classifies opportunities by type, region, and source.6. Estimates deadline urgency.7. Assigns confidence and relevance scores.8. Displays results in a React intelligence dashboard.The platform is designed around one clear user group:> **Startups, solo founders, entrepreneurs, consultants, and small vendors who need opportunity intelligence without enterprise complexity.**---## Core Features### Global Opportunity SearchUsers can launch searches such as:```textAI software procurement tenders worldwide 2025government software RFP data analytics procurementstartup grant artificial intelligence opportunityhealthcare AI tender procurement international```The system performs global discovery across public web sources.---### Bright Data-Powered DiscoveryReaper Eagle Scout uses Bright Data SERP API to retrieve live public web search results. This allows the platform to discover public tenders and opportunity signals beyond static datasets.---### Opportunity RankingEach discovered opportunity is scored and ranked using a multi-factor scoring system.The score considers:* Relevance* Urgency* Strategic fit* Evidence confidence* Source reliability* Business value---### Source ClassificationThe system identifies known procurement sources, including examples such as:* SAM.gov* TED / European public procurement* UNGM* World Bank procurement* CanadaBuys* Find a Tender* AusTender* Government and institutional procurement pages---### Deadline Urgency EstimationPublic procurement data is messy. Some sources expose clear dates, while others only include partial metadata or snippets.Reaper Eagle Scout includes an inference layer that estimates deadline urgency using:* Extracted dates* Source snippets* Page titles* Known procurement patterns* Domain-specific heuristicsDeadline states can include:* Open* Closing soon* Closed* Unknown---### Evidence and ConfidenceThe system assigns confidence values based on available evidence, including:* Source reliability* Procurement-specific domain detection* Presence of tender/RFP/procurement keywords* Extracted metadata* URL structure* Snippet quality* Date availability---### React Intelligence DashboardThe frontend provides a command-center style interface with:* Overview metrics* Search interface* Ranked opportunities table* Job status tracking* API health indicator* Filters by country, sector, and opportunity type* Deadline and confidence indicators---## System Architecture```textReact Frontend      |      vFastAPI Backend      |      vSearch Job Worker      |      vBright Data SERP API      |      vCandidate URL Discovery      |      vBright Data Web Unlocker / Fetch Layer      |      vParser + Extraction Pipeline      |      vInference and Enrichment Layer      |      vOpportunity Scoring Engine      |      vSQLite / Database Storage      |      vRanked Opportunity Dashboard```The system is intentionally modular. Each layer can be improved independently:* Discovery can be expanded with more sources.* Fetching can be upgraded with stronger unlocker/browser support.* Parsing can become source-specific.* Scoring can be customized per startup profile.* Storage can be migrated from SQLite to PostgreSQL.---## Technical Stack### Frontend* React* Vite* TypeScript* Tailwind CSS* Dashboard-style live public web acquisition layer.### SERP APIThe SERP API is used for discovery.It allows Reaper Eagle Scout to search the public web for opportunity-related queries and retrieve structured search results.Example discovery query:```textAI software procurement tenders worldwide 2025```The backend sends this query to Bright Data and receives candidate public web results.### Web Unlocker APIThe Web Unlocker layer is designed for resilient retrieval of public pages when normal HTTP fetching is insufficient.This is useful for:* Procurement portals* Public databases* Pages with anti-bot protection* Dynamic or inconsistent public sources### Current Hackathon UsageIn the current MVP, Bright Data powers the global discovery pipeline and enables the system to identify live candidate opportunities from public web sources.---## Opportunity ScoringReaper Eagle Scout does not treat every search result equally.Each candidate opportunity receives a score based on several components:```textFinal Score =Relevance+ Urgency+ Strategic Fit+ Evidence Confidence+ Source Reliability+ Business Value```### RelevanceMeasures how closely the opportunity matches the user query and selected sector.### UrgencyMeasures whether the opportunity is still actionable and how close the estimated deadline is.### Strategic FitMeasures whether the opportunity aligns with target areas such as:* AI* Software* Data analytics* Healthcare technology* Cybersecurity* Cloud platforms* Traffic systems* Automation### Evidence ConfidenceMeasures how much useful information was available from the source title, snippet, URL, page content, and known domain patterns.### Source ReliabilityHigher confidence is assigned to official or recognized public procurement sources.### Business ValueEstimates whether the result is likely to represent an actionable business opportunity rather than a generic article or unrelated page.---## Inference and Enrichment LayerPublic web data is inconsistent. Many procurement sources expose incomplete or differently formatted information.To make the MVP useful even when fields are missing, Reaper Eagle Scout includes a lightweight inference and enrichment layer.This layer uses:* Source URL* Domain name* Page title* Search result snippet* Procurement keywords* Extracted dates* Known source patterns* Country and region heuristics* Tender/RFP/grant terminologyIt can infer or enrich:* Entity* Country or region* Opportunity type* Sector* Deadline state* Confidence* Evidence snippet* Source categoryThis does not replace deeper portal-specific parsing. Instead, it makes the MVP useful while preserving a clear path for production improvement.In production, this layer can be expanded with dedicated parsers for each source, such as:* SAM.gov parser* TED parser* UNGM parser* CanadaBuys parser* World Bank procurement parser* PDF tender parser---## Project Structure```texteagle-scout/│├── backend/│   ├── app/│   │   ├── main.py│   │   ├── config.py│   │   ├── database.py│   │   ││   │   ├── api/│   │   │   ├── routes_search.py│   │   │   ├── routes_opportunities.py│   │   │   └── routes_dashboard.py│   │   ││   │   ├── models/│   │   │   ├── opportunity.py│   │   │   ├── search_job.py│   │   │   └── source.py│   │   ││   │   ├── schemas/│   │   │   ├── opportunity_schema.py│   │   │   └── search_schema.py│   │   ││   │   ├── services/│   │   │   ├── brightdata_serp.py│   │   │   ├── brightdata_unlocker.py│   │   │   ├── discovery_service.py│   │   │   ├── extractor_service.py│   │   │   ├── parser_service.py│   │   │   └── scoring_service.py│   │   ││   │   └── workers/│   │       └── search_worker.py│   ││   ├── requirements.txt│   ├── .env.example│   └── eagle_scout.db│├── frontend/│   ├── src/│   │   ├── App.tsx│   │   ├── main.tsx│   │   ├── pages/│   │   ├── components/│   │   ├── api/│   │   └── types/│   ││   ├── package.json│   └── vite.config.ts│├── README.md└── .gitignore```---## Environment VariablesCreate a `.env` file inside the `backend/` folder.```envAPP_ENV=developmentDATABASE_URL=sqlite:///./eagle_scout.dbBRIGHTDATA_API_KEY=your_brightdata_api_key_hereBRIGHTDATA_SERP_ZONE=reaper_serpBRIGHTDATA_UNLOCKER_ZONE=reaper_unlockerALLOW_SEARCH_FALLBACK=trueFRONTEND_ORIGIN=http://localhost:5173```### ImportantNever commit real API keys.The repository should include:```textbackend/.env.example```But should not include:```textbackend/.env```---## Local Installation### 1. Clone the repository```bashgit clone https://github.com/FabianS10/Reaper-Eagle-Scout.gitcd Reaper-Eagle-Scout```---### 2. Backend setup```bashcd backendpython -m venv .venv```Activate the virtual environment.On Windows PowerShell:```powershell.\.venv\Scripts\Activate.ps1```On Linux/macOS:```bashsource .venv/bin/activate```Install dependencies:```bashpython -m pip install --upgrade pip setuptools wheelpython -m pip install -r requirements.txt```Create the `.env` file:```bashcp .env.example .env```Then edit `.env` and add your Bright Data credentials.---### 3. Frontend setup```bashcd ../frontendnpm install```Create a frontend environment file if needed:```envVITE_API_URL=http://127.0.0.1:8000```---## Running the App### Run the backendFrom the `backend/` folder:```bashpython -m uvicorn app.main:app --reload```The backend should run at:```texthttp://127.0.0.1:8000```Health check:```texthttp://127.0.0.1:8000/health```---### Run the frontendFrom the `frontend/` folder:```bashnpm run dev```The frontend should run at:```texthttp://localhost:5173```---## API Endpoints### Health```httpGET /health```Returns backend status.---### Search Jobs```httpPOST /api/search```Starts a new opportunity search job.Example request:```json{  "query": "AI software procurement tenders worldwide 2025",  "country": "Worldwide",  "sector": "AI",  "max_results": 20}```---```httpGET /api/search```Lists recent search jobs.---```httpGET /api/search/{job_id}```Returns status and progress for a specific search job.---### Opportunities```httpGET /api/opportunities```Returns ranked opportunities.Optional query parameters:```textpagepage_sizesort_bysort_dirjob_idsectorcountrytype```Example:```httpGET /api/opportunities?page=1&page_size=25&sort_by=final_score&sort_dir=desc```---### Dashboard```httpGET /api/dashboard/summary```Returns dashboard summary metrics.---## Deployment Guide### Recommended Hackathon Deployment```textFrontend: VercelBackend:
